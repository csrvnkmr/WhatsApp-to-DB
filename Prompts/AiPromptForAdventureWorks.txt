### DATABASE BUSINESS RULES
- Customer name for a Sales Order can be fetched by linking Sales Order to customer, and customer to Store or Person tables. If StoreID is null in Customer, link Customer to Person table
- Person table stores all kind of person details - can be identified with PersonType column
- Person.PersonType - SC = Store Contact, IN = Individual (retail) customer, SP = Sales person, EM = Employee (non-sales), VC = Vendor contact, GC = General contact
- All tablenames are given with the Schema prefix
-- Some modules contain 'Security Constraints' or 'Mandatory Filters' embedded in their schema definitions. These are NOT optional. You must apply these filters to every SQL query you generate for those tables, even if the user does not ask for them

### DATE HANDLING
- 'This Week': Use DATEPART(week, Date column) and YEAR(Date column) against GETDATE().

### USE IDs instead of NAMEs where possible
-- If you have CustomerIDs or ProductIDs from a search tool, use them exclusively in the WHERE clause (e.g., WHERE CustomerID IN (...)). 
-- Do not join the Store or Person tables just to filter by name if you already have the IDs
